Preferences AI Concept Validation Gateway

End-to-end data and process flow

From a Discord /validate pitch, the operation generates a free Hermes preview, provisions Preferences AI survey/simulation assets, creates a Stripe Checkout unlock, then fulfills paid users by posting dashboard links back to Discord.

Critical data objects

  • User pitch from Discord
  • Hermes preview JSON
  • Preferences AI survey_content
  • survey_id and simulation_id
  • Stripe Checkout Session metadata

Fulfillment path

  • Stripe confirms payment via webhook
  • server.js verifies signature when configured
  • active_session.json supplies asset IDs
  • Discord receives dashboard unlock links
